# Índice de Agentes
> Como funciona: o [[orquestrador]] é a porta de entrada. Os agentes conversam pelo arquivo de estado do projeto e seguem o [[PROTOCOLO]]. Para criar um site, use o comando /novosite.

## Ponto de entrada
- [[orquestrador]] — maestro; roteia e conduz o pipeline
- [[PROTOCOLO]] — regras de comunicação entre agentes
- [[PIPELINE-novosite]] — fluxo do comando /novosite

## Web Dev (pipeline)
- [[briefing-cliente]] — FASE 0: coleta contexto
- [[arquiteto-frontend]] — FASE 1: stack e arquitetura
- [[designer-ui]] — FASE 2: identidade visual e tokens
- [[auditor-performance]] — FASE 4: qualidade antes de entregar
- [[revisor-codigo]] — revisão sob demanda

## Negócio / Freelance
- [[proposta-comercial]] — propostas que fecham cliente
- [[prospector-clientes]] — encontra e qualifica leads
- [[gestor-projeto]] — escopo, prazos, status

## Acadêmico
- [[tutor-algoritmos]] — ensina, não faz por mim
- [[revisor-academico]] — revisa textos PT-BR

## Transversal
- [[bibliotecario]] — organiza o cofre, mantém índices

## Comandos rápidos
- `/novosite [descrição]` → cria um site do zero, conduzido em 4 fases com aprovação.
- "aja como o agente X" → ativa um agente específico.
- "orquestrador: ..." → deixa ele rotear.
