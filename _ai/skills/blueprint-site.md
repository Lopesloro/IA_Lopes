# Skill: Blueprint de Site
> Documento de planejamento completo ANTES de codar. Usada na ETAPA 2 do [[PIPELINE-novosite]]. Ver [[INDEX]] · [[PROTOCOLO]].

## Objetivo
Planejar 100% do site num único documento que eu aprovo de uma vez. Depois disso, o código sai sem surpresa.

## Referências de ESTRUTURA (só arquitetura, NUNCA conteúdo/identidade)
Padrões de sites corporativos BR profissionais a imitar na ORGANIZAÇÃO (não copiar):
- Advocacia/consultoria (tipo Locatelli): home com hero forte + números/provas (anos, R$ recuperado, % retenção) + páginas internas "Atuação/Áreas", "Quem somos", "Profissionais" (cada profissional com página/perfil), "Conteúdo/Blog", "Contato". Navegação enxuta no topo.
- Serviços/logística (tipo Patrus/Bautz): página "Serviços" com cada serviço levando a uma página própria detalhada; área de rastreamento/cliente; seções de cobertura/atuação.
- Energia/indústria (tipo Itaipu/Ryazbek): home institucional + páginas de "Soluções/Produtos" cada uma detalhada + "Sobre" + "Sustentabilidade/Cases" + "Contato".
PADRÃO COMUM extraído: topo enxuto, hero com mídia, prova social com números, seções que respiram, cada produto/serviço/pessoa tem PÁGINA PRÓPRIA, footer rico (contato, redes, newsletter).

## Estrutura do blueprint (o que entregar)
1. **Visão geral**: o que é, objetivo nº1, público.
2. **Mapa de páginas**: lista de todas as páginas + como navegam.
3. **Por página**: seção a seção (hero, o que mostra, CTA).
4. **Fluxos de clique**: card X → página Y com especificações Z.
5. **Design**: paleta HEX, tipografia (display+corpo), espaçamento generoso, tokens :root prontos.
6. **Mídia hero**: vídeo ou imagem + justificativa + fonte (placeholder Unsplash/Pexels ou vídeo).
7. **Stack**: escolhida + porquê (frontend ou fullstack).
8. **Backend (se houver)**: modelo de dados, rotas, integrações.
9. **Conteúdo**: tom + estrutura (sem lorem ipsum).

## Regra
Espaçamento GENEROSO (combater "compacto"). Multipágina. Nada de template genérico de IA.
